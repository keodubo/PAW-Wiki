## Reglas por iteracion

- Iteraciones de 2 semanas
- Demo de fin de iteracion es **obligatoria** (no presentarse = reprobado)
- Cada iteracion debe incluir:
  1. Incorporacion completa del feedback anterior
  2. Iteracion de flujos previos (complejizar)
  3. Nuevos flujos core (no solo ABMs)

## Entregas intermedias

Si no se cumple lo requerido para un sprint, se pide entrega intermedia la clase siguiente. Cada entrega intermedia puede penalizar con hasta **1 punto**.

## Herramientas

- Seguimiento TPE1: proyecto de Plane provisto por la catedra
- Seguimiento TPE2: proyecto de Pivotal Tracker provisto por la catedra
- Versionado: repositorio Git de la catedra
