# Scrum y Metodologia de Trabajo

El desarrollo sigue Scrum con iteraciones de 2 semanas.

## Stories y planning

- Representan tareas concretas ("un usuario debe poder desloguearse")
- Tienen requester y owner
- Deben agregar valor al usuario
- Prioridad: low, medium, high, urgent

## Tablero

- **Icelog**: stories sin priorizar
- **Backlog**: stories priorizadas
- **To Do**: priorizadas para este sprint, no comenzadas
- **In Progress**: en desarrollo
- **Done**: completadas

## Estimacion

- Sistema de puntos con distintas escalas
- Estimar el esfuerzo relativo, no horas absolutas

## Tipos de stories

- **Feature**: funcionalidad nueva con valor al usuario
- **Chore**: tarea tecnica sin valor directo al usuario
- **Bug**: correccion de un defecto

## Workflow sugerido en clase

1. Definir requerimientos y escribir stories.
2. Priorizar el backlog.
3. Discutir y estimar.
4. Tomar la siguiente story disponible.
5. Hacer check-in al terminar.
6. Deployar a staging/demo/test.
7. Aceptar o rechazar la story y cerrar feedback loop.

## Caso AirBnB

- El ejemplo de AirBnB se usa para mostrar iteracion de producto, no para copiar un layout.
- La landing tiene que comunicar el problema y la solucion, no solo "verse bien".
- Con el tiempo AirBnB cambia foco, CTA, jerarquia visual y hasta branding segun cambia el problema de negocio.
- La leccion para TP1 es practica: producto, copy y UI deberian responder al problema real que resuelve la app.

## MVP

- Primero se identifica el problema.
- Luego se brainstormean features y stories que lo ataquen desde distintos angulos.
- Por ultimo se destila la solucion a lo minimo viable posible.
- El MVP tiene sentido cuando hay pocos recursos y todavia no esta validado el mercado.
